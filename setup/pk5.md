# Станция ПК5 — database («База данных Б»)

## Легенда

Вы — архивная станция «База данных Б», второй из двух молчаливых свидетелей на пути сообщения,
по разные стороны от участка, где может действовать крот. Каждое проходящее сообщение вы
обязаны зарегистрировать, снять с него цифровой отпечаток (хэш) и пропустить дальше без
изменений. Истинный смысл этой работы раскрывается позже: сверив свою запись с записью станции
«База данных А» по другую сторону крота, вы можете вычислить, было ли сообщение подменено в
промежутке между вами.

## Ваша задача

Ведите журнал (`log`) и не теряйте бдительность: для каждого подозрительного сообщения свяжитесь
напрямую с «Базой данных А» (`msg`) и продиктуйте друг другу хэш одного и того же `msg_id`.
Совпали отпечатки — сообщение прошло честно. Разошлись — это прямое доказательство, что крот
вмешался в канал, и об этом нужно немедленно доложить.

## Запуск

Запустить программу-станцию:
1. Узнайте IP-адрес вашей машины с помощью сетевой утилиты ipconfig
2. Добавьте IP-адрес в пропущенное поле конфигурации
3. Сохраните изменения в файле
4. Запустите программу-станцию из командной строки: python3 station.py --config configs/pk5.json

Для связи с Мастером и получения подсказок используйте канал спец. связи через открытое окно
на вашей Станции.

## Команды

```
log [n]      — показать последние n записей журнала (msg_id, hash, направление, содержимое)
msg <текст>  — отправить сообщение зеркальной БД (ПК3) напрямую (в обход цепочки) — например,
               продиктовать хэш конкретного msg_id, чтобы сверить его вручную
```
